# View Binding
## Ключевые моменты

+ Связывание представлений - механизм взаимодействия с представлениями, безопасный по отношению к типам. Он эффективнее `findViewById()` и с меньшей вероятностью приведет к ошибкам на стадии выполнения.
+ Связывание представлений может использоваться в коде активностей и фрагментов.
+ Связывание представлений генерирует класс связывания для каждого файла макета приложения.
+ Каждый класс связывания включает свойство для каждого представления в макете, обладающего идентификатором.
+ Используя связывание представлений с активностью, вы задаете значение его свойства `binding` в методе `onCreate()` активности.
+ Используя связывание представлений с фрагментом, вы задаете значение его свойства `binding` в методе `onCreateView()` фрагмента и возвращаете ему значение `null` в `onDestroyView()`.
+ Фрагменты содержат методы жизненного цикла, связанные с состоянием активности.


## Теория
+ Для взаимодействия с представление в коде активности или фрагмента нужно получить ссылка на него вызовом `findViewById()`.
+ `findViewById()` ищет представление в иерархии представлений.
+ Принцип работы:
  + Файл макета заполняется преобразуясь в иерархию объектов `View`;
  + Android ищет в иерархии представление с подходящим идентификатором;
  + Android возвращает объект `View` с искомым идентификатором, ближний к верху иерархии, и преобразует его к типу, заданному при вызове `findViewById()`.
+ Недостатки `findViewById()`:
  + Удлинение кода и затруднение его чтения;
  + Неэффективность при глубокой иерархии;
  + Небезопасность в отношении `null` (недействидельный идентификатор);
  + Небезопасность в отношении типов (ошибка при преобразовании к необходимому типу).
+ `findViewById()` выдает ошибки на стадии выполнения.
+ Связывание представлений используется для обращения к представлению.
+ Связывание представлений - это часть Android Jetpack.
+ При связывании представлений Android не нужно искать подходящий объект `View` в иерархии, просто используется объект связывания.
+ Компилятор предотвращает исключения `null`-указателей и приведение типов на стадии компиляции.
+ При обращении к представлениям с использованием объекта связывания компилятор знает, какие представления доступны и к каким типан они относятся.
+ Связывание представлений - безопасная по отношению к типам, более эффективная альтернатива для вызова `findViewById()`.
+ Когда подключается связывание представлений, для каждого файла макета в приложении автоматически создается класс связывания.
+ Имя класса связывания соотвествует `ИмяМакетаBinding`.
+ Каждый класс связывания включает свойство для каждого представления в макете, обладающее идентификатором.
+ Представления макетов ассоциируются со свойствами класса связывания.
+ Связывание представлений несколько усложняет отдельные части кода.
+ Можно запретить генерировать класс связывания для конкретного файла макета с помощью добавления элемента `tools:viewBindingIgnore="true"` в коренвое представление.
+ Идентификатор представление при использовании View Binding:
  + В `XML` коде - `snake_case`;
  + В коде `Kotlin` - `camelCase`.


# Описание кода Kotlin и XML
## Используемый код
+ Получение ссылки на представление с помощью `findViewById()`:  
  `имяСсылки = findViewById<ТипПредставления>(R.id.идентификатор_представления)`
+ Получение ссылки на представление с помощью `ViewBinding`:  
  `binding.идентификаторПредставления`
+ Подключение связывания представлений в `build.gradle.kts` приложения:  
  ```
  android {
  
    buildFeatures {
        viewBinding = true
    }
  }
  ```
+ Добавление связывания представлений в активность или фрагмент:  
  `import имя.пакета.приложения.databinding.ИмяМакетаBinding`
+ Добавление свойства связывания:  
  `private lateinit var binding: ИмяМакетаBinding`
+ Создание объекта связывания представлений в активности (метод `onCreate()`):  
  `binding = ИмяМакетаBinding.inflate(layoutInflater)`
+ Отображение макета в активности, используя View Binding (метод `onCreate()`):  
  ```
  val view = binding.root
  setContentView(view)
  ```
+ Обработка нажатий на представление с использованием View Binding:   
  ```
  binding.идентификаторПредставления.setOnClickListener {
    // Обработка нажатия
  }
  ```
